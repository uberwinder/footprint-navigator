Add OpenAI and Anthropic as AI providers to modelRouter, add cost tracking, add three preset modes, and add a full model selector UI in Navigator Settings.

IMPORTANT: Remove groq-8b (llama-3.1-8b-instant) entirely from the codebase. It produced inaccurate answers. The minimum model is now groq-70b.

STEP 1 — INSTALL NEW PACKAGES:
In artifacts/footprint-pdf/server/package.json add and install:
- openai
- @anthropic-ai/sdk

STEP 2 — UPDATE modelRouter.ts:

ADD PROVIDER CALL FUNCTIONS:

async function callOpenAI(apiKey, systemPrompt, userContent, model, timeoutMs):
- Use openai package
- model options: "gpt-4o-mini" or "gpt-4o"
- messages: [{role: "system", content: systemPrompt}, {role: "user", content: userContent}]
- max_tokens: 512
- Apply withTimeout
- Throw if empty response

async function callAnthropic(apiKey, systemPrompt, userContent, model, timeoutMs):
- Use @anthropic-ai/sdk
- model options: "claude-haiku-4-5" or "claude-sonnet-4-5"
- Use messages.create with system and user message
- max_tokens: 512
- Apply withTimeout
- Throw if empty response

ADD THREE PRESET MODES:

FREE mode — zero API cost, Groq only:
- simple → groq-70b
- moderate → groq-70b
- complex → groq-70b
- Fallback: gemini-flash if Groq fails

BALANCED mode (default) — smart cost/quality tradeoff:
- simple → groq-70b
- moderate → claude-haiku
- complex → claude-sonnet
- Fallback chain: if primary fails try next tier up

BEST mode — highest quality regardless of cost:
- simple → claude-haiku
- moderate → claude-sonnet
- complex → gpt-4o
- Fallback: claude-sonnet if gpt-4o fails

Each model falls back automatically if it fails or times out.
